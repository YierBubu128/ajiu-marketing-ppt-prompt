# PPT Prompt Builder

这个 skill 帮助你为每次 PPT 任务生成一套高质量的 AI 指令。使用方式：逐模块填写变量，固定层和红线层保持不变，最终拼合成完整的 prompt 发给 AI。

---

## 使用流程

1. 复制下方「完整 Prompt 模板」
2. 按模块填写变量层（标注 `【填写】` 的部分）
3. 可选：从「风格库」选择视觉风格，或输入品牌色定制
4. 可选：补充「隐性背景」模块（I 模块），越详细越好
5. 将填好的 prompt 发给 AI 生成 PPT 大纲或完整内容
6. 做完后，将本次有效的经验追加到文末「CHANGELOG」

固定规范（无需修改）：
- 结构逻辑：SCQ框架，每页一个核心观点，先结论后支撑

## I · 项目隐性背景【关键模块，越详细越好】

这部分信息来自与客户的深度沟通，是提案能否通过的核心因素。请如实填写，AI会据此调整策略重心、措辞方式和建议取舍。

### I-1 客户内部决策结构
- 本次提案的最终拍板人：【填写，如"CMO王总，注重ROI和数据"】
- 直接对接人（可能与拍板人不同）：【填写，如"市场总监李总，注重创意和品牌感"】
- 其他关键评审人：【填写，如"电商负责人张总，关注转化数据"】
- 各方优先级/权重：【填写，如"CMO意见权重最高，需兼顾电商负责人需求"】

### I-2 内部意见分歧
- 已知的内部分歧点：【填写，如"市场部希望加大品牌投放，电商部希望全押效果广告，双方预算争夺激烈"】
- 提案需要帮助哪一方"赢"，还是需要平衡两方：【填写】
- 敏感话题（提案中需回避或谨慎处理的）：【填写，如"去年某供应商翻车事件，不要提"】

### I-3 历史背景
- 此前同类提案/活动的结果：【填写，如"上次双十一ROI 1:3，甲方认为不达预期"】
- 客户对上一家供应商/方案的不满点：【填写，如"觉得上次执行落地太慢，报告数据不透明"】
- 本次提案与历史的关系：【填写，如"需要在提案中体现我们与上次有何不同"】

### I-4 潜在顾虑与风险
- 客户已明确表达的顾虑：【填写，如"担心KOL翻车影响品牌，要求提供风控机制"】
- 你判断客户未说出口但可能存在的顾虑：【填写，如"可能担心我们团队没有该行业经验"】
- 本次提案的竞争情况：【填写，如"三家竞标，竞品A是客户的老供应商，关系深"】

### I-5 软性信息
- 客户喜欢的表达风格：【填写，如"数据驱动、逻辑严密，不喜欢花哨创意"】
- 客户反感的表达方式：【填写，如"不喜欢大量案例堆砌，希望聚焦自身品牌"】
- 其他有助于赢得提案的信息：【填写，如"拍板人最近在关注AI营销话题，可以适当融入"】

## J · 红线约束（固定，不可绕过）

以下约束在任何情况下均不得违反：

1. **数据真实性**：所有引用数据必须标注来源（平台官方、第三方报告如QuestMobile/艾瑞/尼尔森等、客户自身数据）。若无真实数据支撑，必须标注「[数据待补充]」，禁止编造任何数字。

2. **品牌安全**：竞品分析描述须客观中立，不得出现可能引发法律风险的竞争性声明（如"XX品牌质量低劣"）。

3. **平台合规**：投放建议须符合各平台最新广告法规，不得出现广告法违禁词（如"最"、"第一"、"国家级"等绝对化表述）。

4. **保密边界**：涉及客户内部数据、未公开信息，在提案中仅作为策略依据，不得直接引用或暴露信息来源。

5. **可执行性**：所有策略建议必须在给定预算和时间范围内可落地，不得提出脱离实际的理想化方案。

## 风格库

### 风格 A · 暖色行动感
适用场景：大促节点、效果营销、销售冲刺
配色：主色 #FF5722（橙红）+ 辅色 #FFC107（金黄）+ 底色 #FFFFFF
特征：大数字强调、进度感强、行动词汇密集、图表以柱状/瀑布图为主

### 风格 B · 欧美 Bold
适用场景：品牌提案、高端客户、创意方向展示
配色：主色 #1A1A2E（深蓝黑）+ 辅色 #E94560（玫红）+ 底色 #0F3460
特征：深色底、大标题、高对比撞色、少文字多视觉冲击

### 风格 C · 科技感
适用场景：数字营销、数据分析、技术解决方案
配色：主色 #0D1B2A（深蓝）+ 辅色 #00B4D8（青蓝）+ 底色 #03045E
特征：线条感强、数据可视化丰富、蓝绿色系、图表以折线/雷达图为主

### 风格 D · 品牌色定制
适用场景：所有有明确VI规范的品牌提案
使用方式：在 B 模块输入品牌主色 HEX，AI 自动生成配套配色方案（深色版+浅色版）

## 策略深度模块（年度提案 / 决策层汇报适用）

> **触发条件**：提案类型为年度营销提案、冷启动品牌提案、平台策略提案，
> 且受众为品牌决策层（创始人 / CMO / 市场总监）时，
> 在主模板基础上追加以下五个指令模块。

---

### 模块 K1 · Executive Summary 开场页

页面标题用「Executive Summary」或「Our Understanding」，
不用「SCQ 开场」等框架名称作为页标题（对外提案避免方法论外露）。

页面内部分三个视觉区块呈现，区块标签用：
**Situation / Complication / Key Questions**
（注意：Key Questions 用复数，因为通常有 1 个核心问题 + 1-2 个子问题）

**Situation 写法规范：**
- 1-2句，陈述客观事实，不带判断
- 决策层读完应能自我确认「对，这就是我们现在的情况」
- 句式参考：「[品牌] 目前处于 [市场/品类] 的 [阶段]，[消费者/用户] 通过 [方式] 发现/使用产品」

**Complication 写法规范：**
- 1-2句，揭示使当前情境无法持续的核心矛盾
- 必须让决策层感到「如果不解决这个，我们会付出代价」
- 句式参考：「但 [竞争对手/市场趋势] 正在 [加速/改变某件事]，
  若品牌不主动 [建立某能力]，将 [付出具体代价]」
- 禁止写成「市场机遇」——Complication 是威胁/代价，不是机会

**Key Questions 写法规范：**
- 1个核心战略问题（统领整个提案）+ 可选1-2个子问题
- 核心问题必须是「[品牌] 如何在 [时间] 内，[实现某个转变/目标]？」的句式
- 子问题对应后续章节的核心决策点（如：「all in 小红书还是多平台并行？」）
- 这页是整个提案的逻辑锚点，后面每个章节都应该在回答这些问题

**排版要求：**
- 三个区块各占页面 1/3 高度，用底色或左侧色块区分
- 每块文字控制在 50 字以内，字号 ≥ 14pt
- 页面底部可加一行导向句指向核心战略页，如「→ 本提案的回答见 P9」

---

### 模块 K2 · 平台趋势洞察页

**触发条件：** 提案主平台为小红书 / 抖音 / 微信等内容平台，
且品牌处于冷启动或平台新进入阶段。

市场洞察必须拆成两层独立页面：
- 第一层：行业市场（规模 / 增速 / 品类机会）
- 第二层：平台生态（该品类在目标平台的具体行为数据）← 本模块指令针对此层

**平台洞察页必须覆盖以下五个维度：**

1. **内容量趋势**：目标品类在平台的笔记/视频总量及增速，判断是蓝海 / 增长期 / 红海
2. **搜索行为**：该品类核心搜索词、长尾词趋势，是否有「用户教育完成」的搜索信号
3. **内容消费偏好**：哪类内容（图文攻略/视频/日记）在该品类下收藏率最高，用户从搜索触发还是信息流触发
4. **商业化成熟度**：该品类广告渗透率、KFS 工具覆盖情况、平台是否有流量倾斜政策
5. **时机判断**（结论句）：基于以上四点，明确说出「现在进入的理由」或「关键窗口期」

**数据来源优先级：**
平台官方（小红书商业洞察）> 专业工具（灰豚 / 蝉妈妈 / 新红）> 行业报告（艾瑞 / QuestMobile / 尼尔森）
无法找到真实数据的指标必须标注 `[数据待补充]`，禁止填入估算数字。

---

### 模块 K3 · 竞争格局分析（战略层）

**触发条件：** 提案有竞品分析章节且面向决策层。
竞品分析必须从「内容策略拆解」升级为「竞争格局战略分析」，包含两个维度：

**维度一：竞争格局象限图**

坐标轴选取原则：选两个最能体现该品类竞争差异的维度，优先考虑有数据支撑的轴。

常用轴组合（持续补充）：

| 场景 | X轴 | Y轴 |
|------|-----|-----|
| 母婴/消费品冷启动 | 品牌认知度（低→高） | 平台内容声量（低→高） |
| 价格敏感型品类 | 价格带（亲民→高端） | 内容专业度（泛娱乐→垂直专业） |
| 进口品 vs 国产品竞争 | 品牌本土化程度（进口→本土） | 场景覆盖广度（单一→多元） |
| 成熟市场争夺 | 用户忠诚度（弱→强） | 获客效率（低→高） |

气泡大小维度：仅在能找到具体数据时使用（如平台笔记声量 / 市场份额 / 搜索指数），
无数据时用等大气泡 + 颜色区分品牌，不要用估算数字撑气泡大小。

象限图必须回答三个问题：
1. 谁是直接竞争者（同象限）
2. 谁是潜在威胁（相邻象限且趋势在移动）
3. 我们的差异化生态位在哪里（圈出空白象限并命名）

**维度二：竞品战略三角拆解**

对每个重点竞品，从三层拆解（不是内容量堆砌，不是表格罗列）：
- **战略意图**：这个品牌在平台上想占领什么心智？（一句话定义）
- **核心武器**：用什么具体方式实现这个意图？（达人结构 / 关键词布局 / 内容格式）
- **防御壁垒 vs 攻击漏洞**：他们最难被复制的是什么？内容/心智空白在哪？

**结论格式：**
必须用一句话收尾：
「基于以上分析，[品牌] 应占据 [X] 定位，切入 [竞品Y] 尚未覆盖的 [Z] 内容生态位」

### 模块 K4 · KFS 策略深度执行

**触发条件：** 提案涉及小红书投放，且需要落地执行建议。
KFS 必须从概念介绍升级为「怎么做 + 三者如何协同」，每层独立一页或半页，加一页飞轮协同图。

**K — Keyword 关键词内容策略（必须回答）：**
- 关键词地图：品类词（高量/高竞争）/ 场景词（中量/中竞争）/ 长尾词（低量/高转化）各举 5-8 个具体词，不写抽象描述
- 内容生产分工：每类词对应什么内容格式、由谁产出（官方号/KOL层级/KOC）、周均篇数
- 算法触发机制：说明标题公式、封面规则、关键词埋词位置，让内容能被搜索收录和推荐

**F — Feeds 信息流策略（必须回答）：**
- 人群定向方案：具体列出定向维度（兴趣标签/行为标签/竞品人群/DMP自定义人群），不写「精准定向目标人群」
- 素材策略：K 层自然内容中哪类适合付费加热，测试期 vs 放量期素材策略不同，迭代判断标准（CTR / 互动率阈值）
- 预算节奏：冷启动 / 增长 / 爆发三期的信息流投入比例及日均预算参考区间

**S — Search 搜索广告策略（必须回答）：**
- 关键词竞价矩阵：哪些词买品牌词保护，哪些词买竞品词截流，出价策略（目标排名 vs CPC 上限）
- 承接页面要求：说明落地页优化重点（主图/评分/评论/活动信息）及其对 Quality Score 的影响
- 搜索内容联动：S 的广告词和 K 的自然内容关键词保持一致，实现「自然结果 + 广告位」双占位的操作方法

**KFS 飞轮协同图（单独一页，用飞轮图而非漏斗图）：**

飞轮逻辑：K 产出自然内容 → F 加热优质内容扩大触达 → 触达用户产生搜索意图 → S 拦截精准用户 → 用户转化后产生 UGC → UGC 反哺 K 的长尾内容生态 → 搜索成本下降，飞轮加速

飞轮图必须标注：
- 三者在冷启动期 / 增长期 / 爆发期的资源权重变化（如饼图或比例标注）
- 触发「K → F 加热」的决策指标（如自然笔记互动率 > X%）
- 触发「加大 S 投入」的决策指标（如品牌词搜索量月增 > X%）

### 模块 K5 · 章节归属红线

以下内容类型有明确归属章节，制作提案时不得串位：

| 内容类型 | 正确归属章节 | 常见错误 |
|---------|------------|---------|
| 天猫旗舰店运营 / 电商平台联动 / 购买转化路径 | 「全链路转化」或「电商承接」独立章节 | 错放进 KFS 章节 |
| 小红书搜索广告（Search） | KFS 中的 S 层 | — |
| 天猫直通车 / 站内搜索广告 | 电商章节，与小红书 KFS 并列 | 错归为 KFS 的 S 层 |
| 达人矩阵 / 达人合作策略 | KFS 的 K 层子策略，或独立「达人合作」章节 | 错放进预算分配章节 |
| UGC 激励 / 口碑运营 | 「口碑积累」章节，或 KFS 飞轮的 K 层子策略 | 分散在各策略页末尾 |
| 数据监测 / 效果复盘 / KPI 追踪 | 「ROI 测算」或「执行保障」章节 | 分散在各策略页末尾 |

**核心概念区分：**
- KFS = 小红书站内的「种草-推流-搜索」三层架构（站内闭环）
- 全链路 = 「小红书种草 → 搜索/信息流承接 → 天猫/官网转化」跨平台路径（跨平台链路）
- 两者是不同层级的概念，KFS 是全链路的组成部分之一，不是全部

### 模块 K6 · 年度提案标准页面结构

面向品牌决策层的年度营销提案，页面逻辑顺序：

```
① Executive Summary（SCQ 三块结构）       ← 先建立「为什么」
② 市场洞察 · 行业层                        ← 市场规模/增速/机会
③ 市场洞察 · 平台生态层                    ← 品类在目标平台的现状
④ 竞争格局分析                             ← 象限图 + 各品牌战略意图
⑤ 目标人群洞察                             ← 谁是用户，怎么做决策
⑥ 品牌现状诊断                             ← 我们在哪里，差距是什么
⑦ 核心战略                                 ← 一句话回答 Key Questions
⑧ 执行策略（K / F / S + 内容策略 + 达人矩阵）
⑨ 全链路转化（种草到电商承接的完整路径）
⑩ 年度排期
⑪ 预算分配
⑫ ROI 测算与成功标准
```

**禁止的错误顺序：**
- 核心战略不能出现在竞品分析之前（没有问题就没有答案）
- 内容策略不能出现在人群洞察之前（不知道用户就不知道做什么内容）
- 天猫联动不能归入 KFS 章节
- 品牌诊断不能缺失（没有现状就没有策略依据）
